# MVP 验收清单

本文档用于第一版桌面翻译工具的实机验收。

## 自动验证

在仓库根目录运行：

```powershell
dotnet build .\TranslatorTool\TranslatorTool.csproj
dotnet run --project .\TranslatorTool.Tests\TranslatorTool.Tests.csproj
```

当前自动测试覆盖：

- 翻译调度：AI 快响应、AI 慢响应、AI 失败保留离线结果
- Windows OCR 基础识别
- 选中文本服务的剪贴板兜底路径
- 美式音标重点词提取
- 悬浮窗朗读按钮调用朗读服务
- SQLite 历史记录保存、倒序读取、清空

## 启动与退出

- 启动程序后主窗口默认隐藏。
- 系统托盘出现“窗口翻译工具”图标。
- 托盘菜单可打开截图翻译、历史记录、设置。
- 托盘菜单“退出”后程序进程消失。
- 退出后再次启动不报快捷键或鼠标钩子占用错误。

## 场景 1：网页选中文字翻译

1. 打开浏览器页面。
2. 选中一段英文文本。
3. 按住 `Alt`，点击鼠标右键。
4. 预期：不会弹出浏览器右键菜单。
5. 预期：悬浮窗出现在鼠标附近。
6. 预期：显示原文、译文、音标、注释、来源。
7. 预期：点击“复制”后剪贴板为译文。
8. 预期：点击“朗读”后 Windows TTS 朗读原文。
9. 预期：打开历史记录窗口后能看到这条记录，来源为 `Alt + 鼠标右键`。

## 场景 2：剪贴板保护

1. 先复制一段中文到剪贴板，例如 `原剪贴板内容`。
2. 在浏览器或文本编辑器中选中英文。
3. 按 `Alt + 鼠标右键` 翻译。
4. 翻译完成后粘贴剪贴板。
5. 预期：粘贴内容仍为 `原剪贴板内容`。

## 场景 3：截图 OCR 翻译

1. 打开英文网页、PDF 或图片。
2. 按 `Alt + Q`。
3. 拖动框选英文区域。
4. 预期：松开鼠标后触发 OCR。
5. 预期：识别到文本后显示悬浮翻译窗。
6. 预期：打开历史记录窗口后能看到这条记录，来源为 `Alt + Q 截图 OCR`，并标记为 OCR。

## 场景 4：截图取消与空结果

1. 按 `Alt + Q`。
2. 按 `ESC`。
3. 预期：截图遮罩关闭，不弹出翻译窗。
4. 再次按 `Alt + Q`，框选空白区域。
5. 预期：显示“OCR 未识别到文字”的友好提示。

## 场景 5：AI 慢响应或未配置

1. 保持 `appsettings.json` 中 `EnableAiTranslation` 为 `true`，但不配置 `AiApiKey`。
2. 触发一次翻译。
3. 预期：先显示离线翻译结果。
4. 预期：最终来源显示 `AI 翻译失败，当前为离线结果`。
5. 预期：历史记录只保存一条最终结果，不保存重复的临时离线结果。

## 场景 6：历史记录

1. 连续执行一次选中文字翻译和一次截图翻译。
2. 打开托盘菜单“历史记录”。
3. 预期：记录按时间倒序显示。
4. 选中一条记录后点击“复制译文”。
5. 预期：剪贴板变为该条译文。
6. 点击“清空历史”。
7. 预期：列表清空，重启程序后仍为空。

## 当前已知边界

- 离线翻译仍是轻量兜底词典，不是完整离线翻译引擎。
- 在线 AI 翻译需要用户在设置中配置 API Key。
- Windows OCR 依赖系统 OCR 能力和语言包。
- Windows TTS 取决于系统已安装语音。
