# Windows 桌面翻译工具开发需求

## 一、项目目标

开发一个 Windows 桌面翻译工具。

这个工具不是普通的大窗口翻译软件，而是后台常驻工具，主要用于阅读英文网页、PDF、软件界面、Datasheet、GitHub、技术文档时快速翻译。

核心交互方式：

1. 选中文字后，按 `Alt + 鼠标右键`，直接翻译选中的文字。
2. 按 `Alt + Q`，开启截图框，框选屏幕文字区域后 OCR 识别并翻译。
3. 翻译结果以悬浮窗显示在原位置附近。
4. 翻译结果包含中文翻译、美式音标、注释、朗读按钮。
5. 支持离线翻译和 AI 翻译。
6. 如果 AI 翻译 1 秒内没有响应，先显示离线翻译结果；AI 返回后自动覆盖离线结果。
7. 每次翻译都保存历史记录。

## 四、核心功能 1：Alt + 鼠标右键翻译选中文字

### 触发方式

用户在任意软件中选中文字后：

```text
按住 Alt + 鼠标右键
```

程序检测到该操作后，自动获取当前选中的文字并翻译。

### 获取选中文字的逻辑

优先级如下：

1. 优先使用 Windows UI Automation 获取当前选区文本。
2. 如果 UI Automation 获取失败，则使用剪贴板兜底方案。
3. 剪贴板兜底方案流程：
   - 先保存当前剪贴板内容。
   - 模拟 `Ctrl + C`。
   - 等待短时间，例如 100ms 到 300ms。
   - 读取剪贴板文本。
   - 恢复原剪贴板内容。
4. 如果仍然没有获取到文字，则显示提示：
   - “未检测到选中文本，请重新选择文字或使用 Alt + Q 截图翻译。”

### 翻译窗口显示位置

翻译结果悬浮窗显示在鼠标附近。

要求：

1. 窗口置顶。
2. 不抢占输入焦点。
3. 可以手动关闭。
4. 超出屏幕边界时自动调整位置。
5. 窗口不要太大，适合快速查看。

## 五、核心功能 2：Alt + Q 截图 OCR 翻译

### 触发方式

用户按：

```text
Alt + Q
```

进入截图框选模式。

### 截图流程

1. 显示全屏半透明遮罩。
2. 鼠标拖动选择矩形区域。
3. 鼠标松开后截取该区域图像。
4. 将截图送入 OCR 模块识别文字。
5. 将识别出的文字送入翻译流程。
6. 翻译结果显示在截图区域附近，尽量接近原文字位置。

### OCR 要求

第一版可以先抽象接口，不要把 OCR 和业务逻辑写死。

定义接口：

```csharp
public interface IOcrService
{
    Task<string> RecognizeTextAsync(Bitmap image, CancellationToken cancellationToken);
}
```

第一版可以先使用 Windows OCR 或临时 Mock OCR，后续再替换为 PaddleOCR / ONNX OCR / AI OCR。

## 六、翻译结果内容

每次翻译结果需要包含以下内容：

1. 原文
2. 中文翻译
3. 美式音标
4. 注释
5. 朗读按钮
6. 复制按钮
7. 翻译来源

显示示例：

```text
Current loop bandwidth

美式音标：
current /ˈkɜːrənt/
loop /luːp/
bandwidth /ˈbændwɪdθ/

翻译：
电流环带宽

注释：
指电机控制中电流环能够响应电流变化的频率范围。

来源：
AI 翻译
```

如果当前显示的是离线翻译，来源显示：

```text
来源：离线翻译，AI 更新中...
```

AI 结果返回后，自动更新为：

```text
来源：AI 翻译
```

## 七、美式音标要求

只显示美式音标，不显示英式音标。

处理逻辑：

1. 如果是单词，直接查询该单词的美式音标。
2. 如果是短语或句子，提取重点英文单词，显示这些单词的美式音标。
3. 不需要给整句话生成完整音标。
4. 优先从本地词典数据库查询音标。
5. 本地查不到时，可由 AI 返回音标。
6. 需要给音标模块单独设计接口，方便以后替换数据源。

接口建议：

```csharp
public interface IPhoneticService
{
    Task<List<PhoneticItem>> GetAmericanPhoneticsAsync(string text, CancellationToken cancellationToken);
}

public class PhoneticItem
{
    public string Word { get; set; }
    public string AmericanPhonetic { get; set; }
    public string Meaning { get; set; }
}
```

## 八、朗读功能要求

翻译结果窗口中需要有朗读按钮。

朗读要求：

1. 单词优先使用真人发音。
2. 句子使用高质量 TTS。
3. 在线语音优先。
4. 网络不可用时，使用 Windows 系统语音兜底。
5. 朗读音频可以缓存到本地，避免重复请求。

接口建议：

```csharp
public interface ISpeechService
{
    Task SpeakAsync(string text, string language, CancellationToken cancellationToken);
}
```

第一版可以先实现：

1. Windows 系统 TTS 兜底。
2. 在线 TTS 接口预留。
3. 后续再接入 OpenAI TTS / Azure TTS / 其他高质量 TTS。

## 九、翻译引擎要求

软件同时支持：

1. 离线翻译
2. AI 翻译

### 翻译流程

用户触发翻译后：

1. 启动 AI 翻译请求。
2. 同时准备离线翻译请求。
3. 如果 AI 在 1 秒内返回，则直接显示 AI 翻译。
4. 如果 AI 1 秒内没有返回，则先显示离线翻译。
5. 离线翻译显示后，AI 请求继续等待。
6. AI 返回后，用 AI 翻译结果覆盖离线翻译结果。
7. 如果 AI 最终失败，则保留离线结果，并显示：
   - “AI 翻译失败，当前为离线结果。”

### 接口设计

```csharp
public interface ITranslationService
{
    Task<TranslationResult> TranslateAsync(
        string sourceText,
        string targetLanguage,
        CancellationToken cancellationToken);
}
```

建议实现三个类：

```csharp
OfflineTranslationService
AiTranslationService
TranslationCoordinator
```

其中：

- `OfflineTranslationService` 负责离线翻译。
- `AiTranslationService` 负责联网 AI 翻译。
- `TranslationCoordinator` 负责 1 秒超时、离线先显示、AI 后覆盖的调度逻辑。

### TranslationResult 数据结构

```csharp
public class TranslationResult
{
    public string SourceText { get; set; }
    public string TranslatedText { get; set; }
    public string Notes { get; set; }
    public List<PhoneticItem> Phonetics { get; set; }
    public string SourceLanguage { get; set; }
    public string TargetLanguage { get; set; }
    public string Engine { get; set; } // Offline 或 AI
    public bool IsFinal { get; set; }  // AI结果为最终结果，离线结果为临时结果
}
```

## 十、历史记录要求

每次翻译都保存历史记录。

使用 SQLite。

### 保存内容

每条历史记录包含：

1. 原文
2. 译文
3. 美式音标
4. 注释
5. 源语言
6. 目标语言
7. 翻译来源
   - Alt + 鼠标右键
   - Alt + Q 截图 OCR
8. 翻译引擎
   - Offline
   - AI
9. 翻译时间
10. 是否来自 OCR
11. 朗读音频缓存路径，可为空

### SQLite 表结构建议

```sql
CREATE TABLE IF NOT EXISTS TranslationHistory (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    SourceText TEXT NOT NULL,
    TranslatedText TEXT NOT NULL,
    Notes TEXT,
    PhoneticsJson TEXT,
    SourceLanguage TEXT,
    TargetLanguage TEXT,
    TriggerSource TEXT,
    Engine TEXT,
    CreatedAt TEXT NOT NULL,
    FromOcr INTEGER NOT NULL DEFAULT 0,
    AudioPath TEXT
);
```

### 历史记录界面

第一版做简单列表即可。

要求：

1. 按时间倒序显示。
2. 可以查看原文和译文。
3. 可以复制译文。
4. 可以清空历史记录。

建议第一轮只执行：

```text
P1 到 P4
```

也就是：

1. 创建 WPF 项目基础结构。
2. 实现后台常驻和托盘。
3. 实现 Alt + Q 截图框。
4. 实现悬浮翻译窗口。

等 P1 到 P4 可运行后，再继续 P5 到 P9。
